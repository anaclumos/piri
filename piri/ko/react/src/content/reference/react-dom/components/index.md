---
title: React DOM 컴포넌트
---

<Intro>

React는 모든 브라우저 내장 [HTML](https://developer.mozilla.org/en-US/docs/Web/HTML/Element) 및 [SVG](https://developer.mozilla.org/en-US/docs/Web/SVG/Element) 컴포넌트를 지원합니다.

</Intro>

---

## 공통 컴포넌트 {/*common-components*/}

모든 내장 브라우저 컴포넌트는 일부 props와 이벤트를 지원합니다.

* [공통 컴포넌트 (예: `<div>`)](/reference/react-dom/components/common)

여기에는 `ref` 및 `dangerouslySetInnerHTML`과 같은 React 전용 props가 포함됩니다.

---

## 폼 컴포넌트 {/*form-components*/}

이 내장 브라우저 컴포넌트는 사용자 입력을 받습니다:

* [`<input>`](/reference/react-dom/components/input)
* [`<select>`](/reference/react-dom/components/select)
* [`<textarea>`](/reference/react-dom/components/textarea)

이들은 React에서 특별한 컴포넌트입니다. 왜냐하면 `value` prop을 전달하면 *[제어된 컴포넌트](/reference/react-dom/components/input#controlling-an-input-with-a-state-variable)*가 되기 때문입니다.

---

## 리소스 및 메타데이터 컴포넌트 {/*resource-and-metadata-components*/}

이 내장 브라우저 컴포넌트는 외부 리소스를 로드하거나 문서에 메타데이터를 주석으로 추가할 수 있습니다:

* [`<link>`](/reference/react-dom/components/link)
* [`<meta>`](/reference/react-dom/components/meta)
* [`<script>`](/reference/react-dom/components/script)
* [`<style>`](/reference/react-dom/components/style)
* [`<title>`](/reference/react-dom/components/title)

이들은 React에서 특별한 컴포넌트입니다. React는 이들을 문서의 head에 렌더링하고, 리소스가 로드되는 동안 일시 중지하며, 각 특정 컴포넌트의 참조 페이지에 설명된 다른 동작을 수행할 수 있기 때문입니다.

---

### 사용자 정의 HTML 요소 {/*custom-html-elements*/}

대시가 있는 태그를 렌더링하면, 예를 들어 `<my-element>`, React는 [사용자 정의 HTML 요소](https://developer.mozilla.org/en-US/docs/Web/Web_Components/Using_custom_elements)를 렌더링하려고 한다고 가정합니다. React에서 사용자 정의 요소를 렌더링하는 것은 내장 브라우저 태그를 렌더링하는 것과 다르게 작동합니다:

- 모든 사용자 정의 요소 props는 문자열로 직렬화되며 항상 속성을 사용하여 설정됩니다.
- 사용자 정의 요소는 `className` 대신 `class`를, `htmlFor` 대신 `for`를 사용합니다.

[`is`](https://developer.mozilla.org/en-US/docs/Web/HTML/Global_attributes/is) 속성이 있는 내장 브라우저 HTML 요소를 렌더링하면 사용자 정의 요소로 처리됩니다.

<Note>

[React의 향후 버전은 사용자 정의 요소에 대한 더 포괄적인 지원을 포함할 것입니다.](https://github.com/facebook/react/issues/11347#issuecomment-1122275286)

React 패키지를 최신 실험적 버전으로 업그레이드하여 시도해 볼 수 있습니다:

- `react@experimental`
- `react-dom@experimental`

React의 실험적 버전에는 버그가 있을 수 있습니다. 프로덕션에서는 사용하지 마십시오.

</Note>
